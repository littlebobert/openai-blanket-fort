# 毛布の秘密基地（Blanket Fort）

> グループチャットが、アプリをつくる。

毛布の秘密基地は、グループの会話を、その場で使える小さな共有Webアプリに変える
[Hermes Agent](https://hermes-agent.org)プロジェクトです。友だちがチャットで
アイデアを伝えると、Hermesがアプリを生成し、進捗を報告して、公開リンクを会話へ
返します。

![Blanket Fort ソーシャルカード](public/og.png)

## 90秒で審査

1. [公開LP](https://blanket-fort-hermes.littlebobert.chatgpt.site)でコンセプトと
   グループチャット体験を確認します。
2. [常設の審査用DinnerWheel](https://blanket-fort-hermes.littlebobert.chatgpt.site/demo/dinner-wheel)
   で料理に投票し、ルーレットを回します。TelegramやローカルMacは不要です。
3. [Hermesが実際に生成したDinnerWheel](https://kaycee-soundable-unappeasingly.ngrok-free.dev/apps/dinner-wheel/)
   を開き、共有投票とルーレットを試します。
4. [公開ミニアプリハブ](https://kaycee-soundable-unappeasingly.ngrok-free.dev/)で
   生成済みアプリを確認します。

常設デモはLPと同じ環境で公開されています。ライブ生成アプリとハブは、デモMac、
共有バックエンド、ngrokがオンラインの間利用できます。

## Telegramでの実行証明

![TelegramでHermesがDinnerWheelを生成する様子](public/telegram-build-proof.png)

Hermesは自然言語の依頼を受け、ファイル操作の進捗をTelegramへ送り、英語・日本語
対応の共有DinnerWheelを生成して、その公開リンクを返しました。

## 完成したエンドツーエンド体験

```text
Telegramでアプリを依頼
    ↓
Hermesメッセージング・ゲートウェイ
    ↓
専用ワークスペースでHTML/CSS/JavaScriptを生成
    ↓
共有状態・投票・ルーレットAPIへ接続
    ↓
ngrok経由で公開
    ↓
進捗と完成URLをTelegramへ返信
```

## 実装済み

- Hermes Agent v0.18.2とGPT-5.6 Solを使う実際のTelegramボット。
- ファイル検索・書き込みなどの進捗をチャットへ返すHermesゲートウェイ。
- `generated-apps/<slug>/`だけで作業する、範囲を限定した生成ワークスペース。
- Hermesが実際に生成した、モバイル対応・日英対応のDinnerWheel。
- ユーザーごとの投票、集計、共有状態、ルーレット結果を扱う共通バックエンド。
- 生成アプリ一覧と各アプリを配信する公開ハブ。
- LPに常設した、審査時に外部セットアップが不要なDinnerWheel。
- 夕食ルーレット、日程調整、トーナメントの3つを操作できるプロダクトデモ。
- 端末のUI言語を初期値にする日本語・英語切り替え。

## 設計上の境界

- 生成アプリは専用ディレクトリ内に限定されます。
- APIキーやボットトークンを生成アプリへ埋め込みません。
- 各アプリはパッケージ追加や独自サーバーを使わない、自己完結型のWebアプリです。
- 投票アプリはブラウザごとの安定した匿名IDを使い、同じ人の再投票を更新として扱います。
- ハッカソン版の共有データはローカルJSONへ保存されます。本番版では同じAPI契約を
  クラウドDBへ移せます。

# Blanket Fort

> Your group chat builds apps.

Blanket Fort is a group-first app builder powered by
[Hermes Agent](https://hermes-agent.org). A friend describes something useful
in the chat; Hermes builds a small shared web app, reports its progress, and
returns a public link to the conversation.

![Blanket Fort social card](public/og.png)

## Judge this submission in 90 seconds

1. Open the [public product page](https://blanket-fort-hermes.littlebobert.chatgpt.site)
   to understand the concept and interact with the simulated group-chat flow.
2. Open the
   [permanent judge-ready DinnerWheel](https://blanket-fort-hermes.littlebobert.chatgpt.site/demo/dinner-wheel),
   cast a vote, and spin. It requires no Telegram bot or local demo machine.
3. Open the
   [DinnerWheel actually generated by Hermes](https://kaycee-soundable-unappeasingly.ngrok-free.dev/apps/dinner-wheel/)
   and try its shared voting flow.
4. Optionally inspect the
   [live generated-app hub](https://kaycee-soundable-unappeasingly.ngrok-free.dev/).

The permanent demo is hosted with the landing page. The live generated app and
hub demonstrate the real pipeline while the demo Mac, shared backend, and ngrok
tunnel are online.

## Proof: Hermes built and returned the app

![Hermes generating DinnerWheel from Telegram](public/telegram-build-proof.png)

The screenshot shows the real Hermes Telegram bot receiving a natural-language
request, reporting file operations, and returning the completed bilingual
DinnerWheel link.

## Completed end-to-end flow

```text
Natural-language Telegram request
    ↓
Hermes messaging gateway
    ↓
Generate HTML/CSS/JavaScript in a bounded workspace
    ↓
Connect to shared state, voting, and spin APIs
    ↓
Publish through the ngrok-backed app hub
    ↓
Reply in Telegram with progress and the finished URL
```

This flow was run successfully during the hackathon. The resulting app is the
live generated DinnerWheel linked above.

## What is implemented

- A real Telegram bot running Hermes Agent v0.18.2 with GPT-5.6 Sol through
  OpenAI Codex OAuth.
- Hermes tool progress delivered back into Telegram while the app is built.
- A bounded `generated-apps/<slug>/` workspace governed by a tracked
  [`AGENTS.md`](generated-apps/AGENTS.md) publishing contract.
- A genuinely generated, mobile-friendly, bilingual DinnerWheel.
- One shared backend for generated apps, including:
  - per-user votes and live totals;
  - shared app state;
  - server-selected spin results;
  - static app publishing and app discovery.
- A public generated-app hub served through ngrok.
- A permanent judge-safe DinnerWheel hosted with the product page.
- An interactive landing-page demonstration of three group-created app types:
  DinnerWheel, SlotGrid, and BracketBowl.
- English and Japanese interfaces that default to the device UI language and
  remember the selected locale.
- Responsive layouts, keyboard-accessible controls, and reduced-motion support.

## Why this is different

Most AI app builders start with a blank prompt box and one creator. Blanket
Fort starts with an existing group conversation. Preferences, constraints, and
inside jokes already present in the chat become context for a tool everyone can
use immediately.

Hermes provides the agent runtime: messaging channels, sessions, model access,
tool execution, and progress delivery. Blanket Fort adds the product layer:

- a constrained mini-app workspace;
- a publishing contract for generated apps;
- a shared backend for collaborative state;
- public URLs that return to the conversation;
- a judge-safe hosted fallback.

## Architecture

```text
Telegram
   │
   ▼
Hermes Agent gateway ── OpenAI Codex / GPT-5.6 Sol
   │
   ▼
generated-apps/<slug>/index.html
   │
   ├── GET  /api/apps/<slug>/state
   ├── PUT  /api/apps/<slug>/state
   ├── POST /api/apps/<slug>/votes
   └── POST /api/apps/<slug>/spin
   │
   ▼
Shared Node.js backend ── ngrok public URL ── group members
```

## Safety and scope

- Hermes is instructed to work only inside `generated-apps/`.
- Generated apps are self-contained HTML/CSS/JavaScript with no package
  installation, development server, or arbitrary exposed backend.
- Public apps never contain API keys, bot tokens, or other secrets.
- A stable browser-local ID lets the shared backend update one participant’s
  vote rather than counting repeated taps as new people.
- Runtime-generated apps and state are gitignored; the publishing contract is
  tracked.

## Honest runtime notes

- The product page and judge-ready DinnerWheel are permanently hosted and do
  not depend on Telegram, ngrok, or the demo Mac.
- The live Telegram bot, generated-app hub, shared state, and generated
  DinnerWheel run on the demo Mac and are reachable while its backend and ngrok
  tunnel remain online.
- The hackathon backend stores shared state in a local JSON file. A production
  version would move the same API contract to a durable cloud database.
- The Telegram screenshot remains verifiable evidence of the successful
  end-to-end build if the live demo machine is offline during judging.
